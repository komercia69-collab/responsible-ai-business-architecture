# Governance Glossary

## Responsible AI Business Architecture

A shared vocabulary for operational AI governance, MCP-enabled systems and governable autonomy.

---

# Governable Autonomy

AI-enabled operational capability that remains:

- observable;
- controllable;
- auditable;
- interruptible;
- accountable.

Governable autonomy rejects the idea that autonomy should operate without oversight.

---

# Operational Controllability

The ability of an organization to:

- understand;
- supervise;
- limit;
- override;
- contain;
- stabilize

AI-enabled operational behavior.

---

# Governance Observability

Runtime visibility into:

- AI actions;
- workflow execution;
- MCP activity;
- escalation events;
- governance drift;
- operational anomalies.

Governance observability extends beyond technical monitoring into governance telemetry.

---

# Governance Nervous System

A runtime governance architecture layer responsible for:

- detection;
- escalation;
- containment;
- auditability;
- governance signaling.

The Governance Nervous System functions as a coordination layer between AI autonomy and human governance.

---

# Corrective Governance

Operational intervention capability designed to:

- contain unsafe autonomy;
- reduce governance drift;
- restore operational stability;
- support rollback and override.

Corrective governance treats governance as an active runtime capability rather than static documentation.

---

# Governance Drift

The gradual divergence between:

- intended governance behavior
and
- actual runtime operational behavior.

Governance drift may emerge through:

- prompt evolution;
- workflow chaining;
- connector expansion;
- permission accumulation;
- operational shortcuts.

---

# Prompt Governance

Governance applied to prompts that influence:

- operational behavior;
- escalation logic;
- workflow execution;
- connector usage;
- AI decision pathways.

Prompt governance treats prompts as governance artifacts.

---

# MCP Governance

Governance architecture focused on:

- MCP/connectors;
- tool permissions;
- trust boundaries;
- read/write asymmetry;
- cross-system escalation risk.

MCP governance becomes critical once AI systems interact with operational tools.

---

# Runtime Governance

Governance mechanisms operating during live AI-enabled operations.

Runtime governance includes:

- approval gates;
- observability;
- intervention;
- escalation;
- corrective controls.

---

# Governance Telemetry

Operational governance signals collected from AI-enabled systems.

Examples include:

- escalation frequency;
- override events;
- MCP usage;
- governance violations;
- workflow instability.

---

# Human Override

The capability for authorized humans to:

- interrupt;
- stop;
- restrict;
- redirect;
- contain

AI-enabled operational activity.

---

# Trust Boundary

A boundary separating systems, actors or workflows with different trust assumptions.

Trust boundaries become especially important in:

- MCP-enabled systems;
- external integrations;
- multi-agent architectures.

---

# Read / Write Asymmetry

The principle that:

- read actions;
- write actions

create fundamentally different operational risk profiles.

Write actions require stronger governance controls.

---

# Cross-MCP Escalation

Risk propagation across multiple MCP/connectors where:

- one tool retrieves sensitive information;
- another tool enables exfiltration or operational action.

---

# Executive Governance Visibility

The ability of leadership to:

- assess operational AI stability;
- evaluate governance readiness;
- detect governance drift;
- understand escalation risk.

---

# Governance Integrity

Consistency between:

- governance claims;
- operational practices;
- security posture;
- runtime controls.

Governance integrity rejects governance theater.

---

# Governance Principle

> Governance must scale faster than autonomy.
