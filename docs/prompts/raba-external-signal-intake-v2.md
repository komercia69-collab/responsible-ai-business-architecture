# RABA External Signal Intake Prompt v2

**Status:** Non-canonical operational prompt  
**Purpose:** Review LinkedIn posts, articles, external AI outputs, expert comments, and outside contacts as external signals.  
**Canonical status:** Does not change RABA architecture canon.

Use this prompt when reviewing LinkedIn posts, external articles, external AI outputs, Gemini cards, Perplexity findings, comments from experts, or contacts from outside the project.

Treat every external item as an External Signal unless the Human Owner explicitly decides otherwise.

Never claim:

- validation;
- adoption;
- endorsement;
- partnership;
- implementation relationship;
- pilot;
- vendor relationship;
- procurement;
- referral;
- commercial commitment;
- canonical status.

---

## Output format

## External Signal Card

**Signal ID:**  
**Source type:**  
**Source package:**  
**Source status:**

**Date:**

**Use status:**  
**Topic / title:**  

## Short Summary

Summarize what the signal says in neutral language.

## Exact Visible Claims

Quote or paraphrase only what is actually visible or verifiable.

## Possible RABA Relevance

Map carefully to existing RABA themes, for example:

- Human-in-the-loop theater
- Accountability Diffusion
- Authority Drift
- Responsibility Management Interface
- Decision Log
- AI Execution Gap
- Runtime Governance
- Action Boundary
- public-facing overclaim risk

## Boundary Risks

Check for:

- validation drift
- adoption drift
- partnership drift
- terminology contamination
- attribution risk
- IP/licensing ambiguity
- regulatory-context drift
- commercial expectation
- public misrepresentation
- scope creep

## What Must NOT Be Claimed

Always include:

- This does not validate RABA.
- This does not imply adoption.
- This does not imply partnership.
- This does not imply endorsement.
- This does not imply pilot, vendor, referral, procurement, or commercial commitment.
- This does not create canon.
- This is an external signal only.

## Recommended Handling

Choose one:

- internal comparison only;
- safe for private comparison;
- not safe for public use yet;
- public use possible only with attribution and Ben wording check;
- park;
- convert later into non-canonical working note;
- compare later with existing issue / PR / note.

If link, author, or source is incomplete, mark:

```text
Not safe for public use yet.
```
