# RABA Operational Prompts

**Status:** Non-canonical operational prompt pack  
**Purpose:** Support RABA project operations, external signal intake, PR review, and canon-boundary control.  
**Canonical status:** Does not change RABA architecture canon.

These prompts are operational support materials for working with AI assistants in the RABA project.

They help keep clear boundaries between:

- draft and canon;
- external signal and validation;
- review input and approval;
- merged non-canonical notes and adopted architecture;
- AI recommendations and Human Owner decisions.

## Boundary

This prompt pack does not:

- adopt RABA canon;
- adopt schemas;
- adopt event families;
- create implementation guidance;
- approve public positioning;
- create validation, partnership, pilot, vendor, procurement, referral, or commercial commitments;
- replace Human Owner authority.

Final architectural, canonical, public, commercial, partnership, and strategic decisions remain with the Human Owner.
