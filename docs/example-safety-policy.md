# Example Safety Policy

## Purpose

This repository contains:

- governance examples;
- architecture patterns;
- diagrams;
- prompts;
- workflow concepts;
- MCP-related governance materials.

Examples may influence real-world AI deployments.

This policy exists to reduce unsafe example propagation.

---

# Core Principle

> Illustrative examples must not become accidental unsafe deployment patterns.

---

# Unsafe Content Restrictions

The repository should not intentionally include:

- exposed credentials or secrets;
- hidden API keys;
- deceptive prompts;
- stealth exfiltration examples;
- malicious automation chains;
- destructive workflow examples;
- covert data extraction patterns;
- unsafe write-action automation;
- intentionally misleading governance diagrams.

---

# MCP Governance Safety

MCP/connectors introduce operational risk surfaces.

Examples involving MCPs should:

- clearly identify trust boundaries;
- distinguish read vs write actions;
- describe approval expectations;
- avoid suggesting unrestricted autonomy.

---

# Prompt Safety

Prompts included in the repository should avoid:

- bypass instructions;
- hidden escalation manipulation;
- deceptive role injection;
- unsafe operational automation;
- attempts to override governance controls.

---

# Workflow Governance

Workflow examples should:

- preserve human accountability;
- include escalation paths where appropriate;
- avoid irreversible automation;
- support auditability and observability.

---

# Governance Review Guidance

Contributors are encouraged to evaluate:

- operational consequences;
- governance assumptions;
- escalation visibility;
- trust boundaries;
- misuse potential.

---

# Responsible Governance Principle

> Governance examples influence operational reality and should be treated responsibly.
