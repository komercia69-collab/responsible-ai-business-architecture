## Prototype Rule

Prototypes are not Canon specifications by themselves.

They are visual or operational demonstrations of accepted RABA concepts.

A prototype should become part of the main navigation only after it has:

1. a demo or concrete artifact;
2. a short explanatory document;
3. links to the relevant RABA concepts;
4. a clear relationship to Decision Log or project status.

---

## Current Focus

The current priority prototype is:

```text
Responsibility Review Panel
```

It demonstrates the distinction between:

```text
Baseline Compliance = static policy floor
Runtime Governance = concrete action crossing
```

and implements the rule:

```text
Readiness is declared through accountable role-based action,
not inferred through behavioral monitoring.
```
