# v0.1 Partner Preview

> Partner Preview: Human-AI Governance Pilot Materials

Status: planned release notes for the first partner-facing project snapshot.

## Purpose

This preview packages the project as a partner-readable and AI-readable architecture initiative focused on one practical question:

> How can one real AI-assisted workflow become more visible, governable and accountable before autonomy increases?

## Included in This Preview

### Partner-facing entry path

- `docs/partner-engagement.md` — one-page entry point for potential pilot partners.
- `offers/pilot-partner-call.md` — external invitation for pilot partners.
- `offers/minimal-human-ai-governance-pilot.md` — detailed one-workflow pilot format.
- `docs/share-this-project.md` — copy-ready external sharing package.
- `docs/partner-readiness-backlog.md` — operational maturity and partner-readiness backlog.

### AI-readable project layer

- `PROJECT-CONTEXT-COMPACT.md` — compact AI-assisted project context.
- `llms.txt` — AI-readable reading path and classification guidance.
- `AGENTS.md` — guidance for AI coding agents and repository assistants.
- `docs/ai-review-prompts.md` — ready-to-use AI review prompts.
- `docs/discovery-keywords.md` — search, classification and positioning keywords.

### AI contribution and external review intake

- `docs/ai-contribution-intake.md` — structured mechanism for AI-generated improvement suggestions.
- `.github/ISSUE_TEMPLATE/ai-contribution-suggestion.md` — GitHub issue template for AI suggestions.
- `docs/external-ai-suggestion-intake.md` — guide explaining how and why external AI-assisted reviewers can submit recommendations.

### Human-AI coordination concepts and prototypes

- `docs/prototypes/three-ai-project-coordination-room.md` — prototype operating model for human-led multi-AI project development.
- `docs/concepts/human-ai-operational-coordination-lattice.md` — topology model for roles, responsibilities, handoffs, decision gates and evidence flows.
