# RABA Outreach Materials

**Status:** Draft — Outreach Index  
**Purpose:** Organize external communication materials for RABA  
**Canonical status:** Non-canonical. This index does not define accepted RABA architecture.

## Purpose

This folder contains public-facing drafts that help explain RABA concepts to external audiences.

These materials are intended for:

- LinkedIn posts;
- comments under relevant AI governance discussions;
- direct messages to potential reviewers;
- early pilot conversations;
- external feedback requests.

They are not canonical architecture documents.

They should be treated as communication drafts that translate RABA concepts into accessible external language.

## Suggested Publication Order

Recommended sequence:

1. Comment on multi-AI workflow discussions using `multi-ai-governance-comment.md`.
2. Publish the Active Evidence Interlock post.
3. Link interested people to Issue #71 for feedback.
4. If someone shows serious interest, send the RABA Small Pilot Offer.
5. Record meaningful external feedback in GitHub issues or decision logs only after Human Owner confirmation.

## Governance Boundary

Outreach materials may explain, simplify, and invite discussion.

They must not present draft ideas as accepted architecture unless the idea is already canonical.

They must not imply that external feedback, AI agreement, or consultant agreement is sufficient for canonical adoption.

Final architectural approval remains with the Human Owner.

## Review Checklist Before Publishing

Before publishing an outreach draft, check:

- Is the status clear?
- Does it avoid overstating RABA maturity?
- Does it distinguish draft from canonical architecture?
- Does it preserve the principle: AI may be probabilistic. Responsibility must not be?
- Does it avoid treating multi-AI agreement as approval?
- Does it invite feedback without promising implementation or partnership prematurely?

## Key Message

RABA outreach should make one thing clear:

AI can accelerate preparation, critique, and coordination.

But responsibility must remain human-owned, explicit, reviewable, and recorded.
