# AI Readiness Quick Check

Answer 7 questions before starting your first AI pilot.

This is a fast 3-minute self-check for people, teams, and organizations that want to use AI without losing control, responsibility, or operational stability.

## Why This Exists

Many organizations start AI adoption by asking:

> Which AI tool should we use?

A better first question is:

> Is this business process ready for a responsible AI pilot?

This quick check helps you see whether your AI idea is a promising pilot candidate or whether the process needs more preparation first.

## How to Use It

Choose one business process that you may want to improve with AI.

Examples:

- customer emails;
- invoice processing;
- internal knowledge search;
- reporting;
- document review;
- support ticket triage.

Score each question:

- 0 = no / unclear
- 1 = partly
- 2 = yes / clear

## The 7 Questions

| Question | Score |
|---|---:|
| 1. Is the business process clearly understood by the team? |  |
| 2. Is there a visible business pain AI could reduce? |  |
| 3. Is the process limited enough for a safe first pilot? |  |
| 4. Is the required data clearly identified? |  |
| 5. Could mistakes be detected and corrected by humans? |  |
| 6. Is it clear where human approval is required? |  |
| 7. Is someone accountable for the final result? |  |

**Total: ____ / 14**

## Interpretation

### 12–14 points — Good Pilot Candidate

This process may be suitable for a first controlled AI pilot.

Recommended next step:

- define the AI role;
- avoid autonomous execution at the beginning;
- add a Confirmation Gate where needed;
- define simple success metrics;
- run a limited pilot.

### 7–11 points — Needs Preparation

The process may have AI potential, but the organization should prepare first.

Recommended next step:

- clarify process ownership;
- define data boundaries;
- identify risk points;
- design human review;
- avoid giving AI broad permissions.

### 0–6 points — Not Ready Yet

This process is probably not ready for AI integration.

Recommended next step:

- understand the process better;
- reduce chaos;
- define responsibility;
- improve data quality;
- start with a lower-risk process.

## Immediate Red Flags

Be careful if any of these are true:

- nobody owns the process;
- the process affects payments, employment, legal rights, safety, or sensitive personal data;
- AI would need broad access to confidential systems;
- employees would be expected to approve AI output without context;
- management expects full automation immediately.

If several red flags are present, do not start with autonomous AI.

## What This Quick Check Gives You

In a few minutes, you should know:

1. Whether your AI idea is a reasonable pilot candidate.
2. Whether the process needs preparation before automation.
3. Where responsibility and human control need attention.
4. Whether you should continue with a deeper assessment.

> Ask what AI should be allowed to do, under whose responsibility, with which controls, and for which business value.
