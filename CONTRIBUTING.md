# Contributing

Thank you for your interest in Responsible AI Business Architecture.

This project is an open-source development space for Human-AI Operational Coordination:

> how humans, organizations and AI systems remain accountable, observable, correctable and coordinated when AI begins to act inside real workflows.

The project is not a finished doctrine.

It is a structured space for serious discussion, practical examples, governance patterns, failure patterns, diagrams, standards alignment and proof-layer experiments.

## Useful Contribution Types

Good contributions include:

- real AI-assisted workflow use cases;
- failure patterns;
- governance controls;
- action boundary examples;
- standards mapping improvements;
- diagrams and visual explanations;
- proof-layer examples;
- review of demo code;
- accessibility or security improvements;
- clearer documentation and navigation.

---

## Contribution Principles

Please keep contributions:

- practical;
- specific;
- evidence-aware;
- clearly scoped;
- respectful;
- non-hype;
- careful about compliance claims.

Do not claim that this project certifies legal, regulatory, privacy, security or audit compliance.

---

## Preferred Pull Request Style

Small, focused pull requests are preferred.

A good PR should answer:

```text
What problem does this address?
Which project concept does it connect to?
Which file or section changed?
What should reviewers check?
```

## Key Principle

> AI may be probabilistic. Responsibility must not be.

Before AI acts, define where AI is allowed to act.

Before autonomy scales, make responsibility visible, assignable and correctable.
