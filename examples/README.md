# Practical Examples

This folder contains practical example use cases that show how the Responsible AI Business Architecture framework can be applied to real business processes.

The goal of these examples is to make the framework concrete:

- define the business process;
- describe the proposed AI role;
- identify risks and boundaries;
- clarify human responsibility;
- define review checkpoints;
- produce a preliminary pilot decision.

The examples include both promising pilot candidates and red-signal cases where the responsible recommendation is to stop, redesign, or downgrade the proposed AI role.

## How to use these examples

Start with one example and compare it to your own business process.

Ask:

- Is the process clearly defined?
- What should AI be allowed to do?
- What must AI never do?
- Where is human review required?
- What should be documented?
- Which signal would be appropriate: Green, Yellow, Red, or Grey?

These examples are not final compliance assessments. They are practical learning and pilot-preparation materials.

## Why Red Cases Matter

Responsible AI governance should not only help organizations launch pilots.

It should also help them identify when a proposed AI pilot is not ready, too risky, or should be redesigned before implementation.

A framework that cannot say “do not start yet” is not a governance framework.
